codebuild-ios-mcp — AWS Architecture

Agent-driven iOS build & test on AWS CodeBuild macOS, exposed as four MCP tools through a Bedrock AgentCore Gateway. Optional VPC mode reaches internal Nexus & validation services.

Control flow

1
Agent → Gateway
SigV4-signed MCP over HTTPS (AWS_IAM auth)
2
Gateway → Lambda
assumes GatewayInvokeRole, invokes the tool handler
3
Lambda → CodeBuild
StartBuild on the macOS fleet; reads summary.json + presigned URLs back from S3
4
Build → internal
fleet reaches your Nexus & validation services over the private path
Dashed = data / artifact paths: build source (git clone), artifact upload to S3, and AWS API access via VPC endpoints.
⚠ Cost: the MAC_ARM reserved fleet bills continuously (~$25–30/day per instance, ~24 h minimum lease). Tear it down when idle.
Outside the VPC by design: Amazon S3 is a regional service, reached privately through its gateway endpoint. Nexus & validation services are your own infra (on-prem or a shared VPC), not deployed by this stack.

Gateway is created by register-gateway.sh (no IaC resource yet).
