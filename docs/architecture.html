codebuild-ios-mcp · AWS Architecture

AI agents build and test iOS apps on AWS CodeBuild macOS through six MCP tools, exposed via a Bedrock AgentCore Gateway.

Required core path (code + artifacts)
Optional VPC, endpoints, internal network
How it works
1 Agent requests a test run
  • The agent calls ios_test through the AgentCore Gateway, passing the branch + scheme, and gets a build_id back right away.
2 Gateway forwards to the backend
  • The gateway, the one AWS_IAM-signed door in, hands the call to the Lambda backend.
3 Lambda starts the build
  • Lambda calls CodeBuild StartBuild on the macOS fleet.
4 The Mac builds & tests
  • The fleet clones your code from GitHub, runs the iOS tests on a simulator, and writes results + artifacts to S3.
5 Agent polls for results
  • The agent polls ios_build_status with the build_id until the run finishes, then reads back pass/fail, failing tests, and artifact links.
Add-ons (optional)
A Reach private company systems VPC
  • Run builds inside your VPC so they reach an internal package server (Nexus) and private test services, with no public internet needed.
B Faster re-tests & many apps tune
  • The reserved Mac stays warm between builds, so re-tests recompile only what changed and finish in seconds, and one shared fleet serves many apps.
The MCP tools the agent gets
ios_testStart a build & test run for a branch and scheme; returns a build_id to track it.
ios_build_statusGet a run's status and results by build_id: pass/fail counts, failing tests, and artifact links.
list_schemesList the schemes available on a branch so the agent can pick the right one.
get_test_logsGet the detailed log and screenshot for a single failing test.
get_build_logWatch the live log tail mid-build, or read the raw output when a build fails before tests run (compile or dependency errors).
ios_cancelStop a wrong or runaway build to free the macOS fleet.
⚠ Cost: the macOS build machine bills continuously while it exists (~$25–30/day, ~24 h minimum). Tear it down when idle.
